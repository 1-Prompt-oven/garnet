Data Structures

Hash​

HDEL​

Removes the specified fields from the hash stored at key. Specified fields that do not exist within this hash are ignored. If key does not exist, it is treated as an empty hash and this command returns 0.

Available since:​

1.0

Syntax​

HDEL <key> <field> [field ...]

Return​

Integer Reply:

The number of fields that were removed from the hash, excluding any specified but non-existing fields.

Examples​

> HSET myhash field1 "foo"
(integer) 1

> HDEL myhash field1
(integer) 1

> HDEL myhash field2
(integer) 0

HEXISTS​

Returns if field is an existing field in the hash stored at key.

Available since:​

1.0

Syntax​

HSET <key> <field>

Return​

Integer Reply:

0 if the hash does not contain the field, or the key does not exist. 1 if the hash contains the field.

Examples​

> HSET myhash field1 "foo"
(integer) 1

> HEXISTS myhash field1
(integer) 1

> HEXISTS myhash field2
(integer) 0

HGET​

Returns the value associated with field in the hash stored at key.

Available since:​

1.0

Syntax​

HGET <key> <field>

Return​

Bulk string reply: The value associated with the field.

Nil reply: If the field is not present in the hash or key does not exist.

Examples​


> HSET myhash field1 "foo"
(integer) 1

> HGET myhash field1
"foo"

> HGET myhash field2
(nil)

HGETALL​

Returns all fields and values of the hash stored at key. In the returned value, every field name is followed by its value, so the length of the reply is twice the size of the hash.

Available since:​

1.0

Syntax​

HGETALL <key>

Return​

Array reply: A list of fields and their values stored in the hash, or an empty list when key does not exist.

Examples​


> HSET myhash field1 "Hello"
(integer) 1

> HSET myhash field2 "World"
(integer) 1

> HGETALL myhash
1) "field1"
2) "Hello"
3) "field2"
4) "World"

HINCRBY​

Increments the number stored at field in the hash stored at key by increment. If key does not exist, a new key holding a hash is created. If field does not exist the value is set to 0 before the operation is performed.

The range of values supported by HINCRBY is limited to 64 bit signed integers.

Available since:​

1.0

Syntax​

HINCRBY <key> <field> <increment>

Return​

Integer reply:

The value of the field after the increment operation.

Examples​

> HSET myhash field 5
(integer) 1

> HINCRBY myhash field 1
(integer) 6

HINCRBYFLOAT​

Increment the specified field of a hash stored at key, and representing a floating point number, by the specified increment. If the increment value is negative, the result is to have the hash field value decremented instead of incremented. If the field does not exist, it is set to 0 before performing the operation. An error is returned if one of the following conditions occur:

The field contains a value of the wrong type (not a string). The current field content or the specified increment are not parsable as a double precision floating point number.

Available since:​

1.0

Syntax​

HSET <key> <field> <increment>

Return​

Bulk string reply:

The value of the field after the increment operation.

Examples​


> HSET mykey field 10.50
(integer) 1

> HINCRBYFLOAT mykey field 0.1
"10.6"

HKEYS​

Returns all field names in the hash stored at key.

Available since:​

1.0

Syntax​

HKEYS <key>

Return​

Array reply:

A list of fields in the hash, or an empty list when the key does not exist

Examples​


> HSET myhash field1 "Hello"
(integer) 1

> HSET myhash field2 "World"
(integer) 1

> HKEYS myhash
1) "field1"
2) "field2"

HLEN​

Returns the number of fields contained in the hash stored at key.

Available since:​

1.0

Syntax​

HLEN <key>

Return​

Integer reply

The number of fields in the hash, or 0 when the key does not exist.

Examples​

> HSET myhash field1 "Hello"
(integer) 1

> HSET myhash field2 "World"
(integer) 1

> HLEN myhash
(integer) 2

HMGET​

Returns the values associated with the specified fields in the hash stored at key.

For every field that does not exist in the hash, a nil value is returned. Because non-existing keys are treated as empty hashes, running HMGET against a non-existing key will return a list of nil values.

Available since:​

1.0

Syntax​

HMGET <key> <field> [field ...]

Return​

Array reply

A list of values associated with the given fields, in the same order as they are requested.

Examples​


> HSET myhash field1 "Hello"
(integer) 1

> HSET myhash field2 "World"
(integer) 1

> HMGET myhash field1 field2 nofield
1) "Hello"
2) "World"
3) (nil)

HSET​

Sets the specified fields to their respective values in the hash stored at key.

This command overwrites the values of specified fields that exist in the hash. If key doesn't exist, a new key holding a hash is created.

Available since:​

1.0

Syntax​

HSET <key> <field> <value>

Return​

Integer Reply:

1 if field is a new field in the hash and value was set.

0 if field already exists in the hash and the value was updated.

Examples​


> HSET myhash name "Jon"
(integer) 1

> HGET myhash name
"Jon"
